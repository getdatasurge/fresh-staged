# User Notice Templates

These templates are used to communicate production cutover events to users.

---

## Template 1: Advance Notice (T-48h)

**Subject:** Scheduled Maintenance: FreshTrack Pro System Upgrade - [Date]

**Body:**

Hello FreshTrack Pro Users,

We're writing to inform you of a scheduled system maintenance window for FreshTrack Pro.

**When:**
- **Date:** [YYYY-MM-DD]
- **Time:** [HH:MM] to [HH:MM] [Timezone]
- **Duration:** Approximately 2-4 hours

**What's Happening:**
We're upgrading our infrastructure to improve performance, reliability, and security. This includes:
- Migrating to a new, faster database system
- Upgrading authentication for enhanced security
- Improving monitoring and alerting capabilities

**What You Need to Know:**

1. **System Downtime:** FreshTrack Pro will be unavailable during the maintenance window. You will not be able to:
   - Access the web dashboard
   - Receive alert notifications (SMS/Email)
   - View temperature readings

2. **Sensor Data:** Your temperature sensors will continue to operate normally and store data. All readings during the maintenance window will be uploaded once the system is back online.

3. **Password Reset Required:** For security reasons, you will need to reset your password when logging in for the first time after the upgrade. You'll receive a password reset email at [reset time].

4. **No Data Loss:** All your historical data, sites, alerts, and configurations will be preserved during the upgrade.

**Status Updates:**
During the maintenance window, you can check our status page for real-time updates:
- **Status Page:** https://status.freshtrackpro.com

**Questions or Concerns?**
If you have any questions, please reach out to our support team:
- **Email:** support@freshtrackpro.com
- **Response Time:** We'll respond within 24 hours

Thank you for your patience as we improve FreshTrack Pro!

Best regards,
The FreshTrack Pro Team

## Best Practices

1. **Send during business hours:** Users more likely to read and prepare
2. **Be specific about times:** Include timezone, use 24-hour format if clarity needed
3. **Emphasize password reset:** Users often miss this critical step
4. **Reassure about data:** Explicitly state no data loss
5. **Provide status page URL:** Give users self-service visibility
6. **Keep tone professional but friendly:** Balance urgency with reassurance
7. **Test email templates:** Send test to team before bulk send
8. **Track open rates:** Monitor if users are reading notices
